WHY NANOCOUNT

小小一个，刚好够用。

Nanocount 不追着访客跑，只安静地数好每一次页面打开。怎么部署、怎样保管数据，都听你的。

01

悄悄计数，不打扰

不会偷偷收集 IP、Cookie、User-Agent 或访客身份，只留下真正需要的页面访问数据。

IP 地址 ×访客 Cookie ×设备指纹 ×
02

数据住在自己家

使用 D1、Turso 或本地 SQLite，把网站数据安心放在自己的服务里，想搬家也很轻松。

03

后台也要清清爽爽

浏览、搜索、排序都一目了然。数字偶尔不听话，也可以直接把它改回来。

04

也能和主题好好相处

计数、查询、站点汇总和管理接口都准备好了，主题和小脚本可以轻松接进来。

INSTALL

把脚本轻轻放在网页的 </body> 前。之后的域名、路径和访问次数，就交给 Nanocount 好好记住。

  1. 1
    先安一个小窝Cloudflare、Vercel、Docker 或 Linux 都可以。
  2. 2
    告诉它网站名字用 ALLOWED_HOSTS 只接收自己网站的数据。
  3. 3
    把脚本带回页面刷新一下，第一条来访记录就会出现啦。

DEPLOY

住在哪里，由你来选。

不管把 Nanocount 安在哪里，它都会乖乖做同一件事。以后想换个住处，数据也不会被锁住。

CF推荐

Cloudflare

Worker + D1，轻巧地跑在边缘。免费额度很适合个人网站，还能一键准备好服务和数据库。

一键部署到 Cloudflare
▲Turso

Vercel

从 GitHub 复制一份，再牵上 Turso 数据库。已经住在 Vercel 的网站会很熟悉。

一键部署到 Vercel
▣SQLite

Docker / Linux

一个容器或一个 Node.js 进程就能运行。数据留在本地，打包、备份和搬家都很自在。

查看部署命令

API FIRST

还会和你的小工具说话。

公开接口负责展示计数，管理接口帮你迁移和修正数据。回答很简单，权限也分得清清楚楚。

查看完整 API 文档 →
